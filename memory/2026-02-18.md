## Coze SDK 能力验证

### 背景
叶公询问是否考虑使用 Coze 空间中的云服务配置，验证 Coze SDK 能力。

### 执行内容

1. **读取现有配置**
   - 查看项目技术栈：Python + FastAPI + 阿里云（通义千问、OCR、OSS）
   - 了解项目阶段规划

2. **Coze SDK 能力分析**
   - 读取 Coze SDK README 文档
   - 分析 SDK 类型定义
   - 确认核心功能

### 验证结果

**Coze SDK 支持的功能**:
- ✅ 大语言模型 (LLM) - 支持流式对话、思考链、多模态（图片/视频输入）
- ✅ 语音识别 (ASR) - 语音转文字，最长 2 小时
- ✅ 语音合成 (TTS) - 30+ 音色，支持 SSML
- ✅ 联网搜索 - Web 搜索、AI 摘要、图片搜索
- ✅ 图片生成 - 文生图、图生图、2K/4K
- ✅ 视频生成 - 文本/图片生成视频

**关键发现**:
- Coze LLM 支持多模态输入（ContentPart 可包含 image_url），可通过此能力实现 OCR
- 不适合用 Coze 的存储作为主要 OSS（仅用于生成内容的托管）

### 建议

**推荐方案**: 渐进式迁移

**阶段 1 - 立即集成（新增功能）**:
- 集成 Coze ASR - 语音录入健康数据
- 集成 Coze TTS - 语音播报健康报告
- 集成 Coze 联网搜索 - 健康知识搜索

**阶段 2 - 测试替换（核心服务）**:
- 测试 Coze LLM 健康数据分析质量
- 测试 Coze LLM OCR 识别准确率
- 对比性能和成本

**阶段 3 - 决策**:
- 根据测试结果决定是否完全迁移到 Coze
- 或保持混合方案（Coze 新增功能 + 阿里云核心服务）
