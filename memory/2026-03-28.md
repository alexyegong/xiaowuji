## Evolver 核心功能

### 进化协议 (GEP)
- **Genes (基因)**: 可重用的进化模式和解决方案
- **Capsules (胶囊)**: 成功应用进化的结果记录
- **EvolutionEvents (事件)**: 进化过程的审计日志

### 进化策略
1. **balanced (平衡)**: 50% 创新, 30% 优化, 20% 修复
2. **innovate (创新)**: 80% 创新, 15% 优化, 5% 修复
3. **harden (加固)**: 20% 创新, 40% 优化, 40% 修复
4. **repair-only (仅修复)**: 0% 创新, 20% 优化, 80% 修复

### 安全模型
- ✅ **代码审查**: 审查模式需要人工确认
- ✅ **验证步骤**: 每个变更都有验证命令
- ✅ **回滚机制**: 失败自动回滚到之前状态
- ✅ **范围控制**: 限制每次修改的文件数和行数
- ✅ **禁止自修改**: 保护核心 evolver 代码不被意外修改

## 运行模式

```bash
node index.js --loop
```

## 工作流程

### 单次进化循环
1. **扫描**: 扫描 `memory/` 目录查找日志和错误模式
2. **分析**: 提取信号，匹配现有 Genes
3. **选择**: 选择最佳 Gene 或创建新 Gene
4. **应用**: 生成修复/优化/创新方案
5. **验证**: 运行验证命令确保变更有效
6. **固化**: 记录 EvolutionEvent，更新 Genes/Capsules
7. **报告**: 生成状态报告

### 持续监控模式
- 后台运行，持续监控系统状态
- 每隔一段时间自动运行一次进化循环
- 遇到问题时自动尝试修复
- 可以通过 Hub 接收网络任务

## 技术决策

### 为什么使用 Evolver
1. **标准化**: 使用 GEP 协议确保进化过程的规范性
2. **可审计**: 所有进化步骤都有详细记录
3. **可重用**: 进化经验固化为可复用的 Genes
4. **安全性**: 多层安全检查，防止意外破坏
5. **网络化**: 可以与 EvoMap 网络协作和共享

### 集成策略
- **渐进式**: 先在本地测试，再启用网络功能
- **保守策略**: 使用审查模式确保安全
- **平衡策略**: 使用 balanced 进化策略
- **离线优先**: 核心功能不依赖网络连接
